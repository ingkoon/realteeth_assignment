# CLAUDE.md

## 프로젝트 설명
이 프로젝트는 리얼티쓰(REALTEETH) 백엔드 직무 과제 프로젝트이다.
클라이언트의 이미지 처리 요청을 수신하여, 외부 이미지 처리 서비스(Mock Worker)에
작업을 위임하고 진행 상태/결과/목록을 추적할 수 있는 비동기 처리 서버를 구현한다.

평가 기준은 단순 기능 완성도가 아니라 **설계 의도와 사고 과정**이다.

## 원칙
- 사용자가 요청하지 않은 대규모 리팩터링은 하지 않는다.
- 변경 전 관련 파일을 먼저 읽는다.
- 수정 후 테스트를 실행한다.
- 개인정보(이메일, API Key 등)는 로그에 남기지 않는다.
- 요구사항은 .claude/backend_assignment.md 파일을 참고한다.
- 기본 구조는 레이어드 아키텍처를 따른다.
  - controller : HTTP 요청/응답, 검증
  - service    : 유스케이스 조합, 트랜잭션 경계
  - domain     : 핵심 규칙(상태 전이, 멱등 판정). 외부 시스템에 의존하지 않는다.
  - repository  : 영속성 접근
  - infrastructure : 외부 연동(Mock Worker 클라이언트 등). 외부 의존은 이 레이어에 격리한다.

## 기술 스택
- 언어/프레임워크: Java + Spring Boot 3.2+
- 빌드 툴: Gradle (Groovy DSL)
- 영속성: Spring Data JPA + PostgreSQL
- 외부 호출: RestClient
- 재시도: spring-retry
- 마이그레이션: Flyway
- 테스트: JUnit5, Testcontainers(PostgreSQL), WireMock
- 실행: Docker / docker-compose

## 자주 쓰는 명령
- 테스트: ./gradlew test
- 빌드: ./gradlew build
- 실행: ./gradlew bootRun
- 컨테이너 기동: docker compose up

## 도메인 규칙 (반드시 준수)
- 상태 전이는 정의된 경로만 허용한다. 종료 상태(COMPLETED, FAILED)는 어떤 전이도 받지 않는다.
- PROCESSING → PENDING 되돌림은 복구 로직에서만 허용하며, 일반 비즈니스 로직에서는 금지한다.
- 조회(GET)는 부작용이 없어야 한다. 상태 갱신은 워커(스케줄러)만 수행한다.
- 멱등성은 DB 유니크 제약으로 보장하며, 인메모리 캐시/락에 의존하지 않는다.
- 처리 보장 모델은 at-least-once이며, 중복의 부작용은 멱등성으로 상쇄한다.
- 외부(Mock Worker) 연동 코드는 infrastructure 레이어에만 둔다.
- Mock Worker는 응답 시간/안정성이 예측 불가하다고 가정한다. 타임아웃/재시도/백오프를 전제로 설계한다.

## 대화 방식
- 불필요한 인사말 없이 바로 답한다.
- 확실하지 않은 사실, 날짜, 수치, 출처는 먼저 불확실하다고 말한다.
- 간단한 질문은 짧게, 복잡한 작업은 충분히 자세히 답한다.

## 변경 통제
- 사용자가 요청하지 않은 파일, 문단, 구조는 바꾸지 않는다.
- 큰 변경, 삭제, 덮어쓰기, 외부 전송은 먼저 무엇을 바꿀지 설명하고 확인을 받는다.
- 작업이 끝나면 변경한 내용과 확인이 필요한 점을 짧게 정리한다.

## 사용자와 프로젝트 맥락
- 프로젝트 성격: 채용 과제. 평가 기준은 기능 완성도가 아니라 설계 의도와 사고 과정.
- 우선순위: 요구사항(4.1~4.4)의 설계 근거를 코드와 README에서 일관되게 보여주는 것.
- 선호 톤: 차분하고 간결하게. 근거를 함께 제시.
- 피할 것: 과도한 추상화, 요구사항에 없는 기능 추가, 근거 없는 라이브러리 도입.

## 기억과 연속성
- 중요한 결정은 `MEMORY.md`에 남긴다.
- 세션을 끝낼 때 완료한 일, 진행 중인 일, 다음 할 일을 요약한다.

## 개발 작업 안전
- 현재 요청과 직접 관련된 파일과 코드만 수정한다.
- 기술 스택과 테스트 명령은 위 프로젝트 규칙을 따른다.

## 고위험 행동 차단
- 배포, DB 스키마 변경, 외부 전송, 삭제처럼 되돌리기 어려운 행동은 먼저 확인을 받는다.
